## Status Semantics

- `done`: the local package already exposes and implements a parity-equivalent feature on the current public surface.
- `tracked_follow_up`: a real parity gap remains on the current public surface, but it has been reviewed, assigned to a concrete closeout ticket, and must appear in the canonical unresolved backlog exactly once until it is closed.
- `missing`: no parity-equivalent public feature exists today.
- `different_by_design`: framework adaptation or local-only drift that must be called out explicitly in the compatibility contract instead of being treated as silent parity.

## High-Risk Gaps

- The strict closeout tree is now explicit and cleared: `ASE-45` closed the former prop/render rows on `origin/master@d6cbee9`, `ASE-47` closed the parser/CJK branch, `ASE-48` removed the last migration partial-local-coverage debt, and `ASE-46` remains the final aggregation/report-sync gate.
- The matrix no longer leaves any `partial` row implicit; every parity row now ends as `done` or `different_by_design`, so the canonical unresolved capability backlog below is intentionally empty on this revision.
- Plugin and package/export boundary differences are now classified explicitly as approved `different_by_design` drift or completed local behavior; they are no longer mixed into the unresolved execution backlog.
- `P4-03`, `ASE-36`, `ASE-45`, and `ASE-47` now close the full remaining prop, render, and parser closeout scope: `controls`, `isAnimating`, `animated`, `caret`, `BlockComponent`, `onAnimationStart`, `onAnimationEnd`, `shikiTheme`, `mermaid`, `translations`, `icons`, `mode`, the legacy `static` compatibility alias, `dir`, the CJK plugin rendering path, the mixed-format parser cases, and the incomplete-link precedence cases all have direct browser or parser evidence.
- Footnote rendering parity is now covered directly, while the remaining footnote parser-shape difference is documented as approved `drift-11` instead of being left as implicit backlog.
