## Status Semantics

- `done`: the local package already exposes and implements a parity-equivalent feature on the current public surface.
- `partial`: the local package has a meaningful equivalent, but the public shape, defaults, or behavior still differs, or parity is not yet proven by a dedicated contract/ported test.
- `missing`: no parity-equivalent public feature exists today.
- `different_by_design`: framework adaptation or local-only drift that must be called out explicitly in the compatibility contract instead of being treated as silent parity.

## High-Risk Gaps

- The local security pipeline now includes `rehype-sanitize`, `rehype-harden`, `allowedTags`, `literalTagContent`, and `normalizeHtmlIndentation`, but the frozen `linkSafety` modal/interceptor flow and the upstream `Markdown` filtering surface still remain release blockers.
- Plugin and package/export boundary differences are now classified explicitly as either approved `different_by_design` drift or completed local behavior; they are no longer tracked as implicit implementation backlog.
- `P4-03` closed much of the prop-surface gap for `mode`, `isAnimating`, `animated`, `caret`, `dir`, `lineNumbers`, `linkSafety`, `allowedTags`, `literalTagContent`, `prefix`, `onAnimationStart`, and `onAnimationEnd`, but several of those remain compatibility shims rather than full reference-equivalent behavior.
- The local parser now covers custom-tag and literal-tag preprocessing, but it still has documented drift in mixed-format closure ordering, inline-citation-vs-link recovery, the standalone `RemendOptions` surface, and block-splitting parity.
- Footnotes and component overrides still need a final parity pass even where the local behavior now exists.
