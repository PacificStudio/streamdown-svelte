# Feature Parity Matrix

This matrix tracks parity against the frozen upstream target from [docs/reference-version.md](./reference-version.md):

- Reference repo: `vercel/streamdown`
- Frozen commit: `5f6475139a87dee8af08fcf7b01475292bc064d2` (`5f64751`)
- In-scope packages: `streamdown`, `remend`, `@streamdown/code`, `@streamdown/math`, `@streamdown/mermaid`, `@streamdown/cjk`
- Reviewer decision rules for acceptable differences live in [docs/parity-contract.md](./parity-contract.md).

Audit inputs used for this matrix:

- Reference package surface: `references/streamdown/packages/streamdown/index.tsx`
- Reference parser surface: `references/streamdown/packages/remend/src/index.ts`
- Reference plugin surfaces:
  - `references/streamdown/packages/streamdown-code/index.ts`
  - `references/streamdown/packages/streamdown-math/index.ts`
  - `references/streamdown/packages/streamdown-mermaid/index.ts`
  - `references/streamdown/packages/streamdown-cjk/index.ts`
- Reference behavior evidence: `references/streamdown/packages/streamdown/__tests__/*` and `references/streamdown/packages/remend/__tests__/*`
- Local package surface: `package.json`, `src/lib/index.ts`, `src/lib/context.svelte.ts`, `src/lib/Streamdown.svelte`
- Local parser and rendering evidence: `src/lib/utils/parse-incomplete-markdown.ts`, `src/lib/marked/*`, `src/lib/Elements/*`, `src/tests/*`

## Status Semantics

- `done`: the local package already exposes and implements a parity-equivalent feature on the current public surface.
- `partial`: the local package has a meaningful equivalent, but the public shape, defaults, or behavior still differs, or parity is not yet proven by a dedicated contract/ported test.
- `missing`: no parity-equivalent public feature exists today.
- `different_by_design`: framework adaptation or local-only drift that must be called out explicitly in the compatibility contract instead of being treated as silent parity.

## High-Risk Gaps

- The reference security model (`rehype-sanitize` + `rehype-harden` + `allowedTags` + `literalTagContent` + `linkSafety`) is not implemented on the local public surface.
- The reference plugin contract and official plugin packages are entirely absent; the local package hardcodes built-in math, mermaid, and code behavior instead.
- `P4-03` closes the public prop-surface gap for `mode`, `isAnimating`, `animated`, `caret`, `dir`, `lineNumbers`, `linkSafety`, `allowedTags`, `literalTagContent`, `prefix`, `onAnimationStart`, and `onAnimationEnd`, but several of those remain compatibility shims rather than full reference-equivalent behavior.
- The local parser now covers several high-value `remend` recovery cases, but it is not yet aligned for custom-tag/literal-tag preprocessing, the standalone `RemendOptions` surface, or the remaining unported regression bundles.
- Footnotes, HTML handling, and component override behavior are materially different from the reference implementation.

## Matrix Maintenance Rules

- Any new parity issue must reference one or more matrix IDs from this file.
- Any public-contract change must update the relevant row status in the same pull request.
- Rows marked `partial` or `missing` should not be treated as “close enough”; they require either contract tests, ported tests, or an explicit `different_by_design` decision in the compatibility contract.
